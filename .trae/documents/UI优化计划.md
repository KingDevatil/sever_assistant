# UI 优化计划

## 目标
保持现有的程序操作面板结构和输出面板的显示规则（白色或黑色背景，关键字高亮）不变，优化 UI 设计，让视觉显得更现代、更美观、更简洁。

## 设计原则
1. **简洁优先**：避免过度设计，保持界面清晰
2. **一致性**：使用统一的设计语言和配色方案
3. **现代感**：使用现代化的设计元素，如微妙的阴影、圆角、过渡效果
4. **可读性**：优化字体、间距和对比度

## 配色方案

### 浅色模式
- **主背景**：`#f5f7fa`（淡灰蓝色，更柔和）
- **卡片背景**：`#ffffff`（纯白色）
- **主色调**：`#1890ff`（现代蓝色，用于强调）
- **文字颜色**：`#333333`（深灰色，主要文字）
- **次要文字**：`#666666`（中灰色，次要文字）
- **边框颜色**：`#e8e8e8`（浅灰色边框）
- **悬停背景**：`#f0f5ff`（淡蓝色悬停效果）

### 深色模式
- **主背景**：`#1e1e1e`（保持不变）
- **卡片背景**：`#252526`（稍亮的深色）
- **主色调**：`#1890ff`（保持一致的强调色）
- **文字颜色**：`#ffffff`（白色文字）
- **次要文字**：`#b0b0b0`（浅灰色次要文字）
- **边框颜色**：`#3d3d3d`（深灰色边框）

## 优化内容

### 1. 全局样式设置
在 `init_ui` 方法开头添加全局样式表，统一设置：
- 字体：使用系统默认字体栈，确保跨平台一致性
- 间距：统一的 padding 和 margin
- 圆角：统一使用 6px 圆角

### 2. 左侧面板优化
- **TabWidget 样式**：
  - Tab 标签使用更现代的设计
  - 选中状态使用底部蓝色线条指示
  - 未选中状态使用浅灰色背景
  
- **服务器列表**：
  - 列表项使用卡片式设计
  - 悬停效果更明显
  - 已连接/断开状态使用更柔和的颜色指示

- **指令树**：
  - 分类标题使用更清晰的层级
  - 悬停效果优化

- **按钮样式**：
  - 使用更现代的按钮设计
  - 主按钮（添加）使用蓝色填充
  - 次要按钮（删除）使用红色边框

### 3. 右侧面板优化
- **服务器页签**：
  - 保持现有高度
  - 使用更现代的 Tab 样式

- **指令按钮面板**：
  - 分类标题使用左侧蓝色竖线指示
  - 按钮使用卡片式设计
  - 悬停效果使用淡蓝色背景
  - 按钮间距优化

- **输出面板**：
  - **保持白色/黑色背景不变**
  - **保持关键字高亮不变**
  - Tab 标签样式优化
  - 边框使用更柔和的颜色

### 4. 底部按钮区域优化
- 按钮使用更现代的设计
- 主操作按钮使用蓝色填充
- 次要按钮使用灰色边框
- 按钮间距优化

### 5. 菜单栏优化
- 菜单项悬停效果优化
- 使用更柔和的选中状态

## 实现步骤

### 步骤 1：添加全局样式方法
在 `ServerAssistant` 类中添加 `apply_modern_style` 方法，统一管理所有样式。

### 步骤 2：修改 init_ui 方法
在 `init_ui` 方法末尾调用 `apply_modern_style` 方法应用全局样式。

### 步骤 3：优化左侧面板样式
修改左侧面板的样式表，包括：
- TabWidget 样式
- 服务器列表样式
- 指令树样式
- 按钮样式

### 步骤 4：优化右侧面板样式
修改右侧面板的样式表，包括：
- 服务器页签样式
- 指令按钮面板样式
- 输出面板样式（保持白色/黑色背景）
- 底部按钮样式

### 步骤 5：优化深色模式
修改 `toggle_dark_mode` 方法，确保深色模式下样式一致。

### 步骤 6：测试验证
- 测试浅色模式下的显示效果
- 测试深色模式下的显示效果
- 测试各种交互效果（悬停、点击等）

3. **修改方法**：`toggle_dark_mode()` - 优化深色模式样式
4. **修改位置**：指令按钮面板样式（约 1016-1040 行）
5. **修改位置**：默认指令面板样式（约 676-700 行）

## 注意事项

1. **保持输出面板不变**：输出面板的白色/黑色背景和关键字高亮功能保持不变
2. **保持功能不变**：所有现有功能保持不变，只优化视觉效果
3. **兼容性**：确保在不同分辨率下显示正常
4. **性能**：避免使用过多的阴影和渐变效果，确保界面响应速度
